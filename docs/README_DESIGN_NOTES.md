# README Design Notes

OpenEncoder's README structure borrows proven presentation patterns from mature developer-tool repositories without copying their text.

```text
+-----------------------------+-----------------------------------------------+
| Source                      | Pattern adopted                               |
+-----------------------------+-----------------------------------------------+
| astral-sh/uv                | fast first-screen promise + quickstart path   |
| BurntSushi/ripgrep          | explicit why/why-not caveats and benchmarks   |
| FiloSottile/age             | security-tool boundary and compact examples   |
| sharkdp/bat                 | feature scan, install notes, visual clarity   |
| cli/cli                     | command-oriented CLI docs and support links   |
+-----------------------------+-----------------------------------------------+
```

References:

- https://github.com/astral-sh/uv
- https://github.com/BurntSushi/ripgrep
- https://github.com/FiloSottile/age
- https://github.com/sharkdp/bat
- https://github.com/cli/cli

The README should keep the privacy boundary, quickstart, evidence board, and patent wording above any aspirational roadmap material.
